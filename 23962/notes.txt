Алгоритм метода расчета операционных лимитов по подразделениям и филиалам Банка.
Данный метод предназначен для расчета доступного остатка для продажи(конверсии) наличной валюты.

С 09.04.2022 при проведении операций:

1. покупки наличной валюты (группа операций "-27", коды согласно реестру с НИВ: 01 03 14 60 61) операционный лимит должен увеличиться:
	
	1.1 для подразделения на величину суммы операции с вычетом процента филиала от суммы операции, если по ДТ валютная касса (X = OperSum * LIMITATION_SALE_PERCENT / 100);

	1.2 для филиала на величину процента филиала от суммы операции, если по ДТ валютная касса;

2. продажи наличной валюты (группа операции "-26" коды согласно реестру с НИВ: 02 03) 

	2.1 для подразделения при регистрации операции должна быть выполнена проверка, которая производит системный (автоматический) контроль доступного остатка валюты для продажи:

		2.1.1 если подразделение находится в группе подразделений -3 (DEPT_GROUP_CODE = -3) "Подразделения, в которых НЕ производится проверка лимита на продажу валюты" или если в подразделении НЕ включен режим учета операций продажи валюты (параметр CBR_CCY_SELL_LIMIT) проверка пройдена успешно;
		
		2.1.2 иначе по валюте, выбранной на форме проведения операции и подразделению пользователя рассчитывается доступный остаток: если сумма операции не превышает рассчитаный остаток, то проверка успешно пройдена.

	После успешного выполнения проверки и проведения операции продажи наличной валюты операционный лимит должен уменьшиться на величину суммы операции с вычетом процента филиала от суммы операции, если по ДТ валютная касса (X = OperSum * LIMITATION_SALE_PERCENT / 100);

	2.2 для филиала операционный лимит должен уменьшиться на величину процента филиала от суммы операции, если по ДТ валютная касса;
